The Lab (Docker)

The crackstation toolbox image: John, Hashcat, Hydra, rockyou, rules — all staged.

github.com/bytetheio/crackstation

60-second quick start

Install Docker, then drop into the fully-staged cracking station — hashes, wordlists, rockyou, rules, and the Hydra target are all pre-loaded:

git clone https://github.com/bytetheio/crackstation
cd crackstation
docker compose pull            # or: docker compose build
docker compose up -d vuln-login
docker compose run --rm crackstation

# Your first crack (Challenge 1):
john --format=raw-md5 -w $ROCKYOU /lab/sample-hashes/c1_md5.txt
john --show --format=raw-md5 /lab/sample-hashes/c1_md5.txt
# -> recover the password, wrap it as DC256{...}, submit it at board.0byte.sh

For the C9 web challenge (Dillon's Cookie Shop), use the internet-enabled toolbox:

docker compose run --rm crackstation
hydra -l dillon -P /lab/wordlists/cookie-wordlist.txt cookies.0byte.sh -s 443 \
  https-post-form "/login:username=^USER^&password=^PASS^:F=Invalid credentials"
⚖️ Lab only. Everything here runs against synthetic data we created — toy hashes, fake users, deliberately weak passwords. Dillon's Cookie Shop is an authorized class target. Cracking passwords or logins you don't own or have written permission to test is illegal. Keep it in the lab.

How scoring works

  • Crack a hash → recover the password → that's your flag, wrapped as DC256{...}.
  • Submit flags on the live leaderboard. Correct flag → points.
  • Hints cost points (listed per challenge) — but learning beats hoarding points.
  • Top solo and top team win. The challenge ladder has the details.
